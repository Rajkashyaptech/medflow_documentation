User stories describe system features from the perspective of different roles.

---

## Role: SuperAdmin

### User Story 1
As a SuperAdmin,
I want to create and manage hospitals,
so that multiple hospitals can use the platform independently.

### User Story 2
As a SuperAdmin,
I want to onboard Hospital Admins,
so that hospitals can manage their own staff.

---

## Role: HospitalAdmin

### User Story 3
As a HospitalAdmin,
I want to add and manage doctors,
so that doctors can consult patients digitally.

### User Story 4
As a HospitalAdmin,
I want to manage receptionists and medical staff,
so that hospital operations run smoothly.

---

## Role: Doctor

### User Story 5
As a Doctor,
I want to view my assigned patient visits,
so that I can consult patients efficiently.

### User Story 6
As a Doctor,
I want to create a prescription for a visit,
so that patient treatment is properly documented.

### User Story 7
As a Doctor,
I want to add medicines with predefined types (tablet, syrup, injection),
so that prescriptions are clear and unambiguous.

### User Story 8
As a Doctor,
I want to generate a prescription PDF,
so that patients can receive a printable copy.

---

## Role: Patient

### User Story 9
As a Patient,
I want to retrieve my previous prescriptions,
so that I can continue treatment if I lose my consultation paper.

### User Story 10
As a Patient,
I want to clearly understand medicine dosage and instructions,
so that I take medicines safely.
